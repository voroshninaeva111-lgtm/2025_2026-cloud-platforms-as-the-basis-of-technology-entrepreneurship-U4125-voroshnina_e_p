## Цель работы

Ознакомиться с основными понятиями и принципами работы облачного хранилища, изучить различные модели хранения данных, такие как блочное, файловое и объектное хранилище, а также получить практический опыт работы с сервисом Google Cloud Storage.

В рамках лабораторной работы необходимо было создать Cloud Storage bucket, загрузить в него изображения, создать папку внутри bucket, переместить туда файлы, настроить публичный доступ, получить ссылки на файлы и удалить созданные ресурсы после выполнения задания.


## Используемые сервисы

В ходе выполнения лабораторной работы использовался сервис **Google Cloud Storage**.

Cloud Storage — это облачное объектное хранилище, предназначенное для хранения файлов, изображений, видео, резервных копий, архивов и других объектов. Данные в Cloud Storage размещаются внутри специальных контейнеров, которые называются **buckets**.

В данной работе использовалась модель **объектного хранения**, так как изображения были загружены в bucket как отдельные объекты.

### 2. Создание Cloud Storage bucket

В разделе **Cloud Storage → Buckets** был создан новый bucket.

Название bucket:

`lab3-storage-evoroshnina`

При создании bucket были выбраны следующие параметры:

| Параметр                 | Значение     |
| ------------------------ | ------------ |
| Location type            | Multi-region |
| Location                 | eu           |
| Storage class            | Standard     |
| Access control           | Uniform      |
| Public access prevention | Disabled     |
| Protection               | Soft Delete  |

После создания bucket появился в списке ресурсов Cloud Storage.

### 5. Настройка публичного доступа

Для того чтобы изображения можно было открыть по ссылке, был настроен публичный доступ к объектам bucket.

Во вкладке **Permissions** был добавлен новый принципал:

`allUsers`

Для него была назначена роль:

`Storage Object Viewer`

После сохранения настроек в карточке bucket появился статус:

`Access granted to public principals`

Это означает, что доступ к объектам был предоставлен публичным пользователям.

## Результат работы

В результате выполнения лабораторной работы были выполнены следующие действия:

1. Выбран существующий проект в Google Cloud.
2. Создан Cloud Storage bucket.

## Вывод

В ходе лабораторной работы были изучены основные возможности Google Cloud Storage. Был создан bucket для хранения объектов, загружены изображения, выполнена организация файлов с помощью папки, а также настроен публичный доступ к объектам.

На практике было показано, что Cloud Storage удобно использовать для хранения статических файлов, изображений и других объектов, к которым может быть предоставлен доступ через публичные ссылки.

Также была закреплена важная практика работы с облачной инфраструктурой — удаление созданных ресурсов после завершения лабораторной работы, чтобы избежать лишних затрат и не оставлять неиспользуемые сервисы в проекте.
